Master KyOS Dragon Arch from installation to advanced customization

Introduction to KyOS

KyOS (Dragon Arch) is a powerful, Arch Linux-based distribution designed for developers, security professionals, and power users who demand excellence. Built around the elegant Qtile window manager, KyOS provides a perfect balance of power, performance, and beauty.

What makes KyOS special? KyOS combines the flexibility of Arch Linux with carefully curated defaults, professional security tools, and an intuitive installation experience.

Key Features

  • Qtile Window Manager: Python-configured tiling window management
  • Smart Installer: Automated, dual-boot, and manual installation modes
  • Security Suite: Professional security and penetration testing tools
  • Development Ready: Pre-configured development environment
  • Performance Optimized: Fast boot times and minimal resource usage
  • Beautiful Aesthetics: Professional themes and modern UI

Installation Guide

System Requirements

Minimum

  • 64-bit x86 processor
  • 2GB RAM
  • 20GB free disk space
  • UEFI or BIOS boot support

Recommended

  • Modern multi-core processor
  • 8GB+ RAM
  • 50GB+ SSD storage
  • UEFI with Secure Boot

Download and Prepare

  1. Download KyOS: Get the latest ISO from GitHub Releases
  2. Verify Checksum: Always verify the SHA256 checksum for security
  3. Create Bootable Media: Use tools like Rufus, Etcher, or dd command
  4. Backup Data: Always backup important data before installation

Creating Bootable USB (Linux/macOS)

# Replace /dev/sdX with your USB device
sudo dd if=kyos-2024.12-x86_64.iso of=/dev/sdX bs=4M status=progress
sync

Installation Process

KyOS features an intelligent installer with three modes:

Automated Install

Perfect for new users. One-click installation with smart defaults.

  • Automatic disk partitioning
  • Optimal filesystem selection
  • Standard user setup
  • Essential packages only

Dual Boot

Install alongside existing operating systems safely.

  • Automatic OS detection
  • Safe partition resizing
  • Bootloader configuration
  • Preserved existing data

Manual Setup

Full control for advanced users and specific requirements.

  • Custom partitioning
  • Filesystem selection
  • Advanced bootloader options
  • Package selection

Step-by-Step Installation

  1. Boot from USB: Enter BIOS/UEFI and set USB as first boot device
  2. Start Installer: Run kyosinstall from the live environment
  3. Choose Mode: Select your preferred installation mode
  4. Configure System: Set timezone, locale, and user account
  5. Install System: The installer handles package installation and configuration
  6. Complete Setup: Remove USB and reboot into your new KyOS system

Qtile Window Manager Basics

Qtile is a full-featured, hackable tiling window manager written and configured in Python. It's designed to be simple, small, and extensible.

Core Concepts

  • Tiling: Windows automatically arrange to use all screen space
  • Groups: Virtual workspaces for organizing windows
  • Layouts: Different arrangements for windows (tile, max, floating, etc.)
  • Python Configuration: All settings in readable Python code

Essential Keybindings

Navigation

Super + h/j/k/l Move focus between windows
Super + Tab Cycle through groups
Super + 1-9 Switch to group 1-9

Window Management

Super + Shift + h/j/k/l Move windows
Super + Ctrl + h/l Resize windows
Super + q Close window

Applications

Super + Return Open terminal
Super + r Application launcher
Super + Space Change layout
Customizable Keybindings: All keybindings can be customized in ~/.config/qtile/config.py. See our Keybindings Reference for the complete list.

Troubleshooting

Common Issues and Solutions

Boot Issues

Problem: System won't boot after installation

Solution:

  1. Check BIOS/UEFI boot order
  2. Verify bootloader installation
  3. Use live USB to repair bootloader
# Boot from live USB and repair
sudo mount /dev/sdaX /mnt  # Replace X with root partition
sudo arch-chroot /mnt
grub-install /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg

WiFi Connection Issues

Problem: Cannot connect to WiFi

Solution:

  1. Check if WiFi adapter is recognized: ip link
  2. Use NetworkManager TUI: nmtui
  3. Restart NetworkManager: sudo systemctl restart NetworkManager

Audio Not Working

Problem: No sound output

Solution:

  1. Check audio devices: pactl list sinks
  2. Unmute audio: amixer sset Master unmute
  3. Restart audio service: systemctl --user restart pulseaudio
